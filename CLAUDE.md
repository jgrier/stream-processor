## Project

A Flink-like stream processing library built on top of Restate (Java SDK 2.4.1) for demonstration purposes. Shows how Restate's durable execution, Virtual Objects, and state management can back familiar stream processing abstractions.

## Build Commands

```bash
./gradlew build                          # Build everything
./gradlew :stream-processor-core:build   # Build library only
./gradlew :stream-processor-example:run  # Run the WordCount example
```

## Running the Example

Requires Restate runtime:
```bash
docker compose -f stream-processor-example/docker-compose.yml up -d
./gradlew :stream-processor-example:run
# In another terminal:
restate deployments register http://localhost:9080
curl -X POST http://localhost:8080/SourceOperatorService/ingest
```

## Architecture

Two Gradle modules:
- `stream-processor-core` — the library
- `stream-processor-example` — demo WordCount app

### Pipeline Execution Model

User builds a DAG via fluent API → `execute()` compiles it into Restate service stages:

1. **SourceOperatorService** (`@Service`): `ingest()` reads from Source, sends each record to `process()`. `process()` applies pre-keyBy stateless operators (map/filter/flatMap), routes results by key to the keyed stage or directly to sink.
2. **KeyedOperatorService** (`@VirtualObject`): Each unique key is a separate Virtual Object instance with isolated state. Applies stateful operators (reduce/ProcessFunction) using Restate's durable state, then forwards to sink.
3. **SinkOperatorService** (`@Service`): Writes records to the configured Sink.

Records flow between stages as JSON strings via `ctx.send()` (one-way durable sends). Type information is preserved via `RecordEnvelope` (class name + JSON value).

### Key Mapping: Flink → Restate

- `keyBy()` → Restate Virtual Object key partitioning
- Keyed state (`ValueState`, `ReducingState`) → Restate `StateKey` / `ctx.get()` / `ctx.set()`
- Exactly-once per-key processing → Virtual Object exclusive handler semantics
- Durable operator execution → Restate's durable execution with journal replay

### Packages

- `dev.streamprocessor.api` — User-facing fluent API: `StreamExecutionEnvironment`, `DataStream`, `KeyedStream`
- `dev.streamprocessor.api.functions` — Operator interfaces: `MapFunction`, `FlatMapFunction`, `FilterFunction`, `KeySelector`, `ReduceFunction`, `ProcessFunction`
- `dev.streamprocessor.api.state` — State abstractions: `ValueState`, `ReducingState`, `RuntimeContext`
- `dev.streamprocessor.api.source` / `api.sink` — Pluggable I/O: `Source`, `Sink`, `InMemorySource`, `ConsoleSink`, `CollectorSink`
- `dev.streamprocessor.internal` — Pipeline compilation and Restate integration. The `*Handlers` and `*Client` classes are generated by Restate's annotation processor.

### Static Configuration Pattern

The three Restate services are generic — they accept operator chains as static fields configured by `PipelineDeployer` before the HTTP server starts. This avoids runtime bytecode generation. Supports one pipeline per JVM.
